## SEO Without the Snake Oil

SEO advice online ranges from genuinely useful to outright scam. Claude helps with the useful parts — writing metadata that accurately represents your content, structuring data for search engines, and finding gaps in your content strategy. It will not help you game the algorithm. It will help you make your content more discoverable to people actually searching for it.

## Title and Meta Description Generation

### Generating Titles

```
Generate 5 SEO title options for this blog post:

Post topic: How to migrate from REST to GraphQL incrementally
Primary keyword: REST to GraphQL migration
Target audience: Backend developers

Requirements:
- Under 60 characters
- Include the primary keyword naturally
- Make each option take a different angle (how-to, story, listicle,
  question, contrarian)
```

Claude generates:

```
1. "How to Migrate from REST to GraphQL Without a Rewrite" (52 chars)
   Angle: how-to, emphasizes incremental approach

2. "We Migrated to GraphQL — Here's What Happened" (47 chars)
   Angle: story/experience, curiosity-driven

3. "5 Steps to Migrate REST APIs to GraphQL Safely" (48 chars)
   Angle: listicle, implies structured process

4. "Should You Migrate from REST to GraphQL?" (41 chars)
   Angle: question, appeals to decision-makers

5. "REST to GraphQL Migration: A Practical Guide" (46 chars)
   Angle: straightforward, targets informational intent
```

Each option serves a different search intent. You pick the one that matches your content and audience.

### Meta Descriptions

```
Write a meta description for this post:
- 150-160 characters
- Include primary keyword: "REST to GraphQL migration"
- Communicate the unique value (incremental approach, real experience)
- Include a soft call to action
- Don't start with "In this article" or "This post covers"
```

Claude generates:

```
"Learn how to migrate from REST to GraphQL incrementally — no rewrites
needed. Based on our 6-month migration with real performance data
and lessons learned." (158 characters)
```

## JSON-LD Schema Markup

Schema markup is tedious to write manually. Claude generates it accurately:

### Article Schema

```
Generate JSON-LD Article schema for this blog post:
- Title: "How to Migrate from REST to GraphQL Without a Rewrite"
- Author: Jane Developer
- Published: 2024-03-15
- Modified: 2024-03-20
- Description: [meta description]
- Image: https://example.com/images/rest-to-graphql.jpg
- Publisher: DevBlog (logo at https://example.com/logo.png)
- Word count: 2,100
```

Claude generates:

```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "How to Migrate from REST to GraphQL Without a Rewrite",
  "author": {
    "@type": "Person",
    "name": "Jane Developer"
  },
  "datePublished": "2024-03-15",
  "dateModified": "2024-03-20",
  "description": "Learn how to migrate from REST to GraphQL incrementally...",
  "image": "https://example.com/images/rest-to-graphql.jpg",
  "publisher": {
    "@type": "Organization",
    "name": "DevBlog",
    "logo": {
      "@type": "ImageObject",
      "url": "https://example.com/logo.png"
    }
  },
  "wordCount": 2100,
  "mainEntityOfPage": {
    "@type": "WebPage",
    "@id": "https://example.com/blog/rest-to-graphql-migration"
  }
}
```

## On-Page SEO Checklist

After writing a post, run Claude's checklist:

```
Review this blog post against these on-page SEO criteria:
1. Primary keyword in the title, H1, first paragraph, and at least
   one H2
2. Meta description under 160 characters with keyword
3. Images have descriptive alt text
4. At least 3 internal links to related content
5. At least 2 external links to authoritative sources
6. Heading hierarchy is correct (one H1, logical H2/H3 nesting)
7. URL slug is short and includes the primary keyword
8. Content length is competitive for the target keyword
9. No keyword stuffing (keyword density under 2%)
10. Table of contents for posts over 1,500 words

Primary keyword: "Docker multi-stage builds"
Post: [paste]
```

Claude checks each criterion and flags what is missing or needs adjustment.
